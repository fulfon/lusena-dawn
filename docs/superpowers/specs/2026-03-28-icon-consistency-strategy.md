# PDP Icon Consistency Strategy

*Date: 2026-03-28*

## Goal

Give customers a consistent icon experience across all PDPs that builds trust and maximizes conversion through pattern recognition.

## Principles

1. **One icon = one meaning.** No exceptions across the entire catalog.
2. **Small vocabulary.** 8 variable icons + 3 locked = 11 total. Repetition IS the feature.
3. **Bundle echo rule.** Bundles reuse 2-3 icons from their component products, reinforcing "you're getting both benefits in one."
4. **No copy changes.** Only icon assignments change. All card titles and descriptions remain as-is.

## Technical Requirements

### 1. Create `palette` icon (NEW)

**SVG design:** 24x24 viewBox, stroke-based (consistent with all LUSENA icons). Three small circles arranged in a loose cluster or gentle arc, representing three color swatches. Clean, minimal - not a literal artist palette.

**Animation:** Add to `lusena-icon-animated.liquid` animated set + `lusena-icon-animations.css`. Three circles pulse opacity in sequence (0.5 -> 1.0 -> 0.5), staggered by ~2.3s, over 7-second cycle. Same pattern as `sparkles` (sequential reveal). Three swatches echo three scrunchies. Easing: ease-in-out.

**Schema:** Add `palette` option to the `icon` select in `lusena-pdp-feature-highlights.liquid`.

### 2. Create `feather` animation (existing static icon)

**Animation spec** (already written in Maska 3D product file): Single feather shape. Very slow, gentle float - subtle vertical drift (translateY 0 -> -2px -> 0) with barely perceptible rotation (+/-1 deg) over 8 seconds. The feather seems to hover weightlessly. Easing: ease-in-out.

**Files to update:**
- `lusena-icon-animated.liquid` — add `feather` to animated set string + add animated SVG case
- `lusena-icon-animations.css` — add `lusena-feather-float` keyframes
